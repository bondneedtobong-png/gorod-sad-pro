# Настройка аккаунтов, БД и интеграций (ручные шаги владельца)

Этот раздел сайта (кабинеты, админка, авторизация, аналитика) требует нескольких
бесплатных внешних сервисов. Все ключи — только в переменных окружения (в репозиторий
не коммитятся). Полный список переменных — в `apps/web/.env.example`.

> Где задавать переменные: **Vercel → проект → Settings → Environment Variables**
> (для прода) и в локальном файле `apps/web/.env` (для разработки; он в `.gitignore`).

---

## 1. База данных — Neon Postgres (бесплатно)
1. Регистрация на **neon.tech** (удобно через GitHub).
2. **Create project** → регион поближе (EU Central), имя `gorod-sad`.
3. В **Connection Details** скопируйте две строки:
   - **Pooled connection** (с `-pooler` в хосте) → переменная `DATABASE_URL`.
   - **Direct connection** (без `-pooler`) → переменная `DIRECT_URL` (нужна для миграций).
4. Добавьте обе в Vercel env (и локально в `apps/web/.env`).
5. Применение схемы и наполнение данными (делает разработчик/CI):
   ```bash
   cd apps/web
   npm run db:push     # создать таблицы по схеме Prisma
   npm run db:seed     # залить 15 растений, 6 услуг и админа из env
   ```

## 2. Авторизация — Auth.js
1. Сгенерируйте секрет: `npx auth secret` (или `openssl rand -base64 32`) → `AUTH_SECRET`.
2. `AUTH_URL` — базовый адрес сайта (на проде: `https://<ваш-домен>`; локально `http://localhost:3000`).
3. Админ-доступ: задайте `ADMIN_EMAIL` и `ADMIN_PASSWORD` — из них seed создаёт пользователя с ролью ADMIN.

## 3. Вход через VK (OAuth)
1. **dev.vk.com** → создать приложение (тип «Веб-сайт»).
2. В настройках укажите **Redirect URI**: `https://<ваш-домен>/api/auth/callback/vk`
   (и `http://localhost:3000/api/auth/callback/vk` для разработки).
3. Скопируйте **ID приложения** → `VK_CLIENT_ID`, **Защищённый ключ** → `VK_CLIENT_SECRET`.

## 4. Письма (сброс пароля) — Resend
1. **resend.com** → зарегистрируйтесь, создайте **API Key** → `RESEND_API_KEY`.
2. Подтвердите домен отправителя (или используйте тестовый адрес Resend на старте) и
   задайте `EMAIL_FROM`, напр. `Город-сад <no-reply@ваш-домен>`.

## 5. Аналитика — Яндекс.Метрика
1. **metrika.yandex.ru** → создать счётчик для домена.
2. Номер счётчика → `NEXT_PUBLIC_YANDEX_METRIKA_ID`.
3. Если переменная пустая — Метрика не подключается. Скрипт грузится **только после
   согласия пользователя на аналитические cookies**.

## 6. Юридические страницы и ПДн
- Страницы `/privacy`, `/cookies`, `/terms` содержат **шаблонный** текст с TODO-плейсхолдерами
  (название оператора, ИНН, адрес, email). Заполните реквизиты.
- ⚠️ Тексты — не юридическая консультация. Перед публичным сбором персональных данных
  покажите документы юристу и при необходимости зарегистрируйтесь оператором ПДн в Роскомнадзоре.

## Чек-лист переменных (Vercel + локально)
| Переменная | Откуда | Обяз. |
|---|---|---|
| `DATABASE_URL` / `DIRECT_URL` | Neon | да |
| `ADMIN_EMAIL` / `ADMIN_PASSWORD` | вы | да |
| `AUTH_SECRET` / `AUTH_URL` | сгенерировать / домен | да |
| `VK_CLIENT_ID` / `VK_CLIENT_SECRET` | dev.vk.com | для входа через VK |
| `RESEND_API_KEY` / `EMAIL_FROM` | resend.com | для сброса пароля |
| `OPENROUTER_API_KEY` | openrouter.ai | для ИИ-чата (уже настроено) |
| `NEXT_PUBLIC_YANDEX_METRIKA_ID` | metrika.yandex.ru | для аналитики |
